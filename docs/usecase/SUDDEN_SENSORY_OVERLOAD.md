# Use Case: Escape to a Safe Haven (Urgent Transition)

## 1. Situation
While walking through a shopping district, the user encounters a sudden construction site with heavy drilling. The intense noise triggers acute sensory overload while they are **in the middle of a trip**.

## 2. System Intervention

### A. Immediate Trigger (The Panic Button)
- **User Action**: The user taps the large **Panic Button** on their phone or triggers it via a wearable complication.
- **On-Device AI ("The Guardian")**: Instantly takes over, knowing that network latency is unacceptable in this moment.

### B. Guided Movement to Safety
- **Immediate Sensory Shelter**: The app plays pre-recorded calming audio and initiates rhythmic haptic patterns to ground the user.
- **Urgent Re-routing**: Once the user is grounded, the app identifies the nearest "Safe Haven" (e.g., a quiet library or park) that was pre-identified during the planning phase.
- **Action**: *"There is a quiet library 50 meters behind you. Let's move there now."*

### C. The "Safe Path" Exit
- **Navigation**: The app launches a simplified navigation UI that only shows the path to the Safe Haven, ignoring all other transit data. It uses high-contrast, low-visual-noise directions to avoid further overstimulating the user during this critical movement.

## 3. Outcome
Instead of having a complete meltdown in a crowded area, the user is successfully "escorted" to a quiet location where they can recover and decide how to continue their journey.
