# Use Case: Sudden Sensory Overload (Panic Mode)

## 1. Situation
While walking through a shopping district, the user encounter an unexpected construction site with heavy drilling. The sudden, intense noise triggers a flight-or-fight response and acute sensory overload.

## 2. System Intervention

### A. Immediate Trigger
- **User Action**: The user taps the large, persistent **Panic Button** on their phone or triggers it via a wearable complication.
- **On-Device AI ("The Guardian")**: Instantly takes over, knowing that network latency is unacceptable in this moment.

### B. Grounding Support (Offline)
- **Haptic Guidance**: The phone begins a rhythmic "square breathing" vibration pattern (4s inhale, 4s hold, 4s exhale).
- **Audio Grounding**: A pre-recorded, familiar voice plays: *"You're safe. Follow the vibrations in your hand. Just breathe with me."*
- **UI Simplification**: The screen dims and displays only a calming, expanding circle synchronized with the haptics.

### C. Safe Haven Escape
- **Navigation**: Once the user's heart rate/breathing stabilizes (detected via wearable or manual "I'm ready" tap), the app identifies the nearest "Safe Haven" found during the planning phase.
- **Action**: *"There is a quiet library 50 meters behind you. Let's go there to rest."*

## 3. Outcome
The user is "grounded" before the panic escalates into a full meltdown and is guided to a safe location to recover.
